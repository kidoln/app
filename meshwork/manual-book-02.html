私有知识库

私有知识库是你个人的长期知识沉淀区。学习如何高效地编辑文档、使用 AI 辅助、组织内容，以及可选的 Git 共享配置。

2.1 私有知识库概述

什么是私有知识库

私有知识库是 MeshWork 中属于当前用户自己的数据边界。它是你个人的主知识沉淀区，用于存放：

  • 个人长期笔记和知识积累
  • 研究资料和学习笔记
  • 写作草稿和灵感片段
  • 不应自动进入团队共享的内容
  • 为多个共享空间提供的私有参考素材

关键特性

私有知识库具有以下特性：

  • 独立性：它独立存在，不从属于任何共享空间。不因为加入或退出某个共享空间而迁移。
  • 持久性：默认长期存在，作为你的主私有沉淀区。
  • Git 可选：可以完全本地使用，也可以后续配置远程仓库进行备份。
  • 素材库：可以作为多个共享空间的私有参考素材来源。

与共享空间的关系

私有知识库和共享空间是两个独立的数据边界：

  • 私有知识库存放你不希望自动进入团队共享的内容。
  • 共享空间存放需要与团队协作的内容。
  • 你可以从私有知识库引用内容到共享空间，但需要显式操作。
  • 两者可以各自独立配置 Git，互不影响。
设计理念

MeshWork 的设计原则是先让你建立私有知识库，开始个人写作和整理，再按需加入共享空间。这样可以保持个人内容的私密性，同时为团队协作提供素材基础。

目录结构

私有知识库在磁盘上的建议目录结构：

<private-library>/
├── .meshwork/          # MeshWork 配置和元数据
├── private-knowledge/  # 你的文档和笔记
└── assets/             # 附件和媒体文件
提示

私有知识库的目录结构由 MeshWork 管理，你不需要手动操作这些目录。所有操作都可以在应用界面中完成。

2.2 文档编辑

创建新文档

在私有知识库中创建新文档：

  1. 在侧边栏中选中私有知识库或其中的某个目录。
  2. 点击工具栏中的"新建文档"按钮，或在文档列表中使用右键菜单选择"新建文档"。
  3. 输入文档标题，按 Enter 确认。
  4. 文档会在主内容区打开，你可以开始编辑。

文档格式基础

MeshWork 使用 Markdown 作为文档格式。Markdown 是一种轻量级标记语言，易于阅读和编写：

标题

# 一级标题
## 二级标题
### 三级标题
#### 四级标题

文本格式

*斜体文本*
**粗体文本**
***粗斜体文本***
~~删除线~~

列表

- 无序列表项 1
- 无序列表项 2
  - 嵌套列表项

1. 有序列表项 1
2. 有序列表项 2

链接和图片

[链接文本](https://example.com)
![图片描述](images/photo.png)

代码

`行内代码`

```
代码块
```

图表代码块

MeshWork 可以直接渲染常用图表代码块。对于产品设计、系统架构和流程说明，推荐在 Markdown 中保留可读的图表源码，让文档既能长期保存，也能在预览中显示为图形。PlantUML 的完整语法可以参考 PlantUML 官方网站。

PlantUML 时序图

```plantuml
@startuml
Alice -> Bob: Authentication Request
Bob --> Alice: Authentication Response
@enduml
```

Mermaid 流程图

```mermaid
flowchart TD
  A[想法] --> B[讨论]
  B --> C[沉淀为文档]
```
图表支持

预览和所见即所得编辑会渲染 plantuml、puml、mermaid、graphviz 和 dot 代码块。PlantUML 适合表达时序图、组件关系和技术设计；Mermaid 适合快速绘制流程图和轻量关系图。

深色模式图表适配：PlantUML、Mermaid 和 Graphviz 图表会在深色模式下使用 SVG 反转匹配算法，自动调整最终渲染结果的线条、填充、文字和用户自定义颜色，使同一份图表源码在浅色与深色界面中都保持清晰可读。

引用

> 引用文本

编辑器工具栏使用

编辑器顶部有局部工具栏，提供常用的格式化操作：

  • 标题下拉菜单：设置当前行或选中文本的标题级别。
  • 粗体/斜体：快速切换文本格式。
  • 列表：插入无序列表或有序列表。
  • 代码块：插入代码块。
  • 引用：插入引用块。
  • 链接/图片：插入链接或图片。
快捷键

大多数格式化操作都有快捷键，如 Command+B 粗体、Command+I 斜体。使用快捷键可以大大提高编辑效率。

插入图片、附件与链接

插入图片

  1. 将图片文件拖放到编辑器中，或点击工具栏的图片按钮。
  2. 图片会被复制到私有知识库的 assets 目录。
  3. 编辑器会自动插入正确的 Markdown 图片语法。

插入附件

  1. 将文件拖放到文档中。
  2. 文件会被复制到 assets 目录。
  3. 文档中会插入一个文件链接。

插入文档链接

  1. 在侧边栏中右键点击要链接的文档。
  2. 选择"复制文档链接"。
  3. 在编辑器中粘贴，会自动插入 [文档标题](文档路径) 格式的链接。

文档保存与版本历史

MeshWork 会自动保存编辑器内容，但"保存文件"和"创建 Git 版本"是两个不同概念：

  • 自动保存：编辑内容会按应用设置保存到本地文件，避免因为切换文档或关闭窗口而丢失。
  • 创建版本：未绑定远端仓库时，编辑器工具栏会显示"创建版本"入口。完成一批有意义的修改后，可以手动保存为本地 Git 版本。
  • 同步：绑定远端仓库后，工具栏主按钮会显示为"同步"。同步前会保存当前文档并提交待同步内容，然后与远端拉取和推送。
  • 历史版本：点击工具栏中的历史按钮，可以查看当前文档、资源历史、已删除项目和同步问题，并按需要恢复。
当前策略

私有知识库未绑定远端仓库时，不会因为每次编辑自动生成提交。这样可以避免 Git 历史充满高频编辑噪音。

2.3 右侧边栏 AI 辅助

打开/关闭 AI 助手面板

右侧边栏的 AI 助手可以为你提供智能辅助：

  1. 在编辑文档时，点击工具栏的 AI 助手按钮，或使用快捷键 Command+Option+0。
  2. 右侧边栏会滑出 AI 助手面板。
  3. 再次点击按钮或使用快捷键可以关闭面板。

基础对话操作

在 AI 助手面板中，你可以：

  • 发送消息：在输入框中输入问题或指令，按 Enter 发送。
  • 查看回复：AI 的回复会显示在对话历史中。
  • 继续对话：在 AI 回复后继续输入，保持上下文连续性。
  • 清空对话：点击面板顶部的清空按钮，重新开始对话。

使用 @ 提及智能体

你可以使用 @ 符号提及特定的 AI 智能体：

  1. 在输入框中输入 @ 符号。
  2. 系统会显示可用的智能体列表，如 @claude、@gpt4 等。
  3. 选择你想要的智能体，或继续输入智能体名称。
  4. 输入你的问题或指令。
  5. 发送后，指定的智能体会回复你的消息。
自动会话窗口

提及智能体后，在接下来的两分钟内（可在设置中调整），你继续发送的消息不需要再次 @。系统会自动将这些消息发送给该智能体，适合连续追问或补充背景信息。

使用 # 引用文档内容

你可以使用 # 符号引用私有知识库中的文档，让 AI 基于具体内容进行分析：

  1. 在输入框中输入 # 符号。
  2. 系统会显示私有知识库中的文档列表，支持搜索和过滤。
  3. 选择要引用的文档。
  4. 输入你的问题或指令。
  5. AI 会基于引用的文档内容进行回复。

示例：

#产品需求 请分析这个需求的技术可行性
提示

你可以同时引用多个文档，AI 会综合所有引用的内容进行回答。这对于需要对比多个文档的场景非常有用。

使用 / 执行自定义命令

你可以使用 / 符号触发预定义的自定义命令：

  1. 在输入框中输入 / 符号。
  2. 系统会显示可用的命令列表，如 /summarize、/action 等。
  3. 选择你想要的命令。
  4. 如果有必要，添加额外的参数或上下文。
  5. 发送后，命令会被执行。

自定义命令需要你自己在应用设置中定义。以下是命令样例，供参考：

  • /summarize：总结当前文档内容
  • /action：提取待办事项
  • /improve：改进当前选中文本的写作
  • /translate：翻译选中文本
如何自定义命令

自定义命令需要在"设置 → 文档"页面中配置。每个命令由名称和提示词组成，定义命令被触发时 AI 应该如何响应。配置完成后，在文档右边栏输入 / 即可看到你定义的命令列表。

提示

如果你输入 / 后没有看到命令列表，说明还没有配置任何自定义命令。请先到"设置 → 文档"页面添加命令，然后再使用。

组合使用技巧

三种命令可以组合使用，创建更丰富的对话场景：

  • @claude 请分析 #产品设计 #技术方案 文档中的风险点
  • /summarize 并 @claude 给出改进建议
  • @gpt4 基于 #竞品分析 写一份产品对比报告

AI 提案与修改建议

AI 助手可以向你提供文档修改建议：

  • 提案：AI 会建议对当前文档的修改，包括增删改内容。
  • 预览：你可以预览应用提案后的文档效果。
  • 应用：点击"应用"按钮，将提案的修改应用到文档。
  • 拒绝：点击"拒绝"按钮，忽略当前提案。
AI 边界

在 MeshWork 中，AI 是参与者而不是自动替代创作者的主体。它可以提出建议，但不会未经确认直接修改你的文档。所有修改都需要你显式确认。

2.4 文档组织

文档目录结构

在私有知识库中，你可以创建目录来组织文档：

  1. 在侧边栏中右键点击私有知识库或现有目录。
  2. 选择"新建目录"。
  3. 输入目录名称。
  4. 可以继续在目录中创建子目录。

建议的目录结构

私有知识库/
├── 笔记/
│   ├── 学习笔记/
│   ├── 会议记录/
│   └── 读书笔记/
├── 项目/
│   ├── 项目A/
│   └── 项目B/
├── 草稿/
└── 归档/

文档移动与重命名

你可以随时调整文档的组织结构：

  • 移动文档：在侧边栏中拖拽文档到目标目录。
  • 重命名文档：右键点击文档，选择"重命名"，输入新名称。
  • 移动目录：拖拽目录到其他位置，可以移动整个目录及其内容。

文档树排序

私有知识库的文档树可以按不同字段排序。入口在左侧边栏上半部分的"私有知识区"标题行右侧：点击 + 按钮，打开菜单后选择"排序"。

  • 按名称排序：适合按文件名或目录名快速浏览，默认使用升序。
  • 按修改时间排序：适合查找最近更新或较久未维护的内容。
  • 按创建时间排序：适合按文档和目录的创建先后整理内容。
  • 切换升降序：再次点击当前排序字段，会在升序和降序之间切换。菜单中的箭头会显示当前方向。

排序设置会保存在当前私有知识库中，下次打开工作台时会继续沿用。

搜索与过滤

当文档数量增多时，使用搜索功能：

  • 工作台搜索：使用 Command+F 聚焦左侧搜索框，搜索私有知识库、共享空间文档和讨论内容。
  • 结果打开：点击搜索结果会打开对应文档或讨论，并在侧边栏中定位到对应资源。
搜索技巧

搜索支持模糊匹配和路径搜索。例如输入"笔记/会议"可以快速找到"会议记录"目录下的文档。

拼音搜索支持

MeshWork 的搜索功能支持拼音搜索，让中文用户可以更方便地查找文档：

  • 全拼搜索：直接输入完整的拼音来搜索中文内容。例如输入"zhongwen"可以找到包含"中文"的文档。
  • 无需切换输入法：使用英文输入状态即可搜索中文文档，无需频繁切换输入法。
  • 混合搜索：可以同时混合使用汉字、拼音和英文进行搜索。例如输入"产品xuqiu"可以找到包含"产品需求"的文档。

注意：拼音搜索基于系统为文档建立的拼音索引。对于多音字或生僻字，搜索结果可能需要结合上下文进行筛选。

2.5 私有知识库的 Git 共享

启用/禁用 Git

私有知识库的 Git 功能是可选的：

  • 不绑定远端：完全本地使用；需要保存版本时，手动点击"创建版本"。
  • 绑定远端：选择"设置 → Git"中维护的远端仓库条目，再为当前私有知识库指定分支。同步时会提交待同步内容并与远端交换数据。
建议

私有知识库可以先本地使用。等你需要多设备备份或远端恢复能力时，再绑定远端仓库也可以。

绑定远程仓库

为私有知识库绑定远程仓库：

  1. 先在"设置 → Git"中添加远端仓库条目，并配置 SSH Key 或 HTTPS Token。
  2. 打开私有知识库的设置页面。
  3. 在 Git 设置中选择要绑定的远端仓库。
  4. 填写分支名，或留空使用自动生成的 meshwork/private/... 分支名。
  5. 点击"确定"保存绑定。

同步

绑定远端仓库后，文档工具栏主按钮会切换为"同步"。点击后，MeshWork 会保存当前编辑器内容，提交本地待同步更改，再执行远端拉取和推送。当前远端拉取采用 fast-forward only 策略：远端可以直接前进时自动完成；如果本地和远端发生分叉，系统会提示同步问题，不会静默创建合并提交。

处理合并冲突

当本地和远端发生分叉，或者同一文件出现无法自动判断的修改时，MeshWork 会把资源标记为同步问题。当前安全边界是：可以 fast-forward 的更新自动完成；复杂冲突不会自动改写你的知识内容，需要你查看历史和差异后再决定如何处理。

手动提交策略

未绑定远端仓库时，私有知识库使用手动版本策略：

  • 查看历史：在文档或目录上右键，选择"查看历史"。
  • 创建版本：完成一批相关修改后，点击"创建版本"，附上有意义的版本说明。
  • 回退版本：如果需要，可以回退到以前的版本。
最佳实践

建议把版本点放在真正有意义的阶段，例如完成一篇文章、整理完一组资料、或准备进行较大修改之前。
