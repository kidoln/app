MeshWork 隐私政策

最后更新：2026年6月

概述

MeshWork 是一款本地优先的创作工作台应用。我们深知隐私的重要性，并致力于保护您的数据安全。本隐私政策说明了我们如何处理您的数据。

核心原则：MeshWork 不运营用于接收或保存用户数据的开发者服务器。您的文档、讨论、设置和凭据默认保存在您的设备上，您完全掌控自己的信息。

我们是否收集数据

MeshWork 不会收集您的个人信息、文档内容、搜索历史、浏览历史或凭据。这里的“不收集”是指这些数据不会被发送到我们控制的服务器，也不会被我们用于分析、广告、追踪或商业画像。

MeshWork 在本地保存的数据

1. 本地数据

MeshWork 在您的本地设备上存储以下数据：

  • 您创建的文档和笔记内容
  • 讨论区的消息和话题
  • 应用设置和偏好配置
  • AI 提供商配置；API 密钥存储在 macOS 钥匙串中
  • 网络搜索提供商配置；搜索服务 API 密钥存储在 macOS 钥匙串中
  • Git 仓库配置；令牌、口令和粘贴的 SSH 私钥内容存储在 macOS 钥匙串中
  • 网络代理配置；代理密码存储在 macOS 钥匙串中

这些数据用于提供应用功能，保存在您的设备上。MeshWork 不会将这些数据上传到我们的服务器；仅本地保存到 macOS 钥匙串或本地文件系统不构成开发者收集。

2. 网络活动

当您主动使用 AI、网络搜索或 Git 同步功能时，应用会向第三方服务发送完成该操作所需的请求。这些请求直接发送到您配置或选择的服务提供商：

  • AI 服务提供商（如 OpenAI、Claude 等）
  • 网络搜索服务提供商（如 Brave Search、Serper、Tavily 等）
  • Git 仓库托管平台（如 GitHub）

这些网络请求直接在您的设备和第三方服务之间进行，MeshWork 不充当中介或代理，我们不会记录或访问这些通信的内容。第三方服务如何处理这些数据，受您选择的服务提供商的条款和隐私政策约束。

我们如何使用您的信息

MeshWork 只在您的设备上使用本地数据来提供您请求的功能。您的数据：

  • 仅存储在您的本地设备上
  • 仅用于您在应用中的直接操作
  • 仅在您主动启用时才会通过 Git 同步到您指定的远程仓库
  • 仅在您主动调用时才会发送到您配置的 AI 或搜索服务

数据共享

MeshWork 不会主动与第三方共享您的数据。只有在您明确执行下列操作时，相关数据才会发送到您选择的服务：

  • 您主动选择将数据通过 Git 推送到远程仓库
  • 您主动使用 AI 功能，将内容发送到您配置的 AI 服务提供商
  • 您主动使用网络搜索功能，向搜索服务提供商发送查询请求

在这些情况下，数据传输直接发生在您的设备与您选择的服务之间，MeshWork 不会访问或存储传输的内容。

分析、广告和追踪

MeshWork 不包含广告 SDK，不使用第三方分析 SDK，也不会跨应用或网站追踪您。我们不会出售您的数据。

数据安全

我们采取以下措施保护您的数据安全：

  • 应用保存的 API 密钥、Git 令牌/口令/私钥内容和代理密码存储在 macOS 钥匙串中
  • 服务名称、模型、URL、端口、用户名和界面偏好等非敏感配置仅存储在本地
  • 应用运行在 macOS 应用沙盒中，限制了文件系统访问
  • 网络通信仅用于您明确配置的服务

儿童隐私

MeshWork 不面向 13 岁以下儿童。MeshWork 不会故意收集儿童的个人信息。

隐私政策的变更

我们可能会不时更新本隐私政策。更新后的政策将在本页面发布，并在页顶标注更新日期。

联系我们

如果您对本隐私政策有任何疑问或建议，欢迎通过以下方式联系：

  • 在 GitHub 上提出 Issue 或 Pull Request
  • 通过 App Store 产品页面联系开发者

MeshWork Privacy Policy

Last Updated: June 2026

Overview

MeshWork is a local-first creative workspace application. We understand the importance of privacy and are committed to protecting your data. This privacy policy explains how we handle your data.

Core Principle: MeshWork does not operate developer servers that receive or store user data. Your documents, discussions, settings, and credentials stay on your device by default, and you remain in control of your information.

Whether MeshWork Collects Data

MeshWork does not collect your personal information, document content, search history, browsing history, or credentials. In this policy, "does not collect" means this data is not sent to servers controlled by us and is not used by us for analytics, advertising, tracking, or commercial profiling.

Data MeshWork Stores Locally

1. Local Data

MeshWork stores the following data locally on your device:

  • Documents and notes you create
  • Discussion messages and topics
  • App settings and preferences
  • AI provider configurations; API keys are stored in the macOS keychain
  • Web search provider configurations; search API keys are stored in the macOS keychain
  • Git repository configurations; tokens, passphrases, and pasted SSH private key content are stored in the macOS keychain
  • Network proxy configurations; proxy passwords are stored in the macOS keychain

This data is used to provide app functionality and is stored on your device. MeshWork does not upload this data to our servers. Storing data locally in the macOS keychain or local file system does not mean the developer collects it.

2. Network Activity

When you actively use AI, web search, or Git sync features, the app sends the requests needed to complete that action to third-party services. These requests are sent directly to the service providers you configure or choose:

  • AI service providers (such as OpenAI, Claude, etc.)
  • Web search providers (such as Brave Search, Serper, Tavily, etc.)
  • Git hosting platforms (such as GitHub)

These network requests occur directly between your device and the third-party services. MeshWork does not act as an intermediary or proxy, and we do not log or access the content of these communications. How those third-party services process data is governed by the terms and privacy policies of the service providers you choose.

How We Use Your Information

MeshWork uses local data on your device only to provide the features you request. Your data:

  • Is stored only on your local device
  • Is used only for your direct actions within the app
  • Is synced to remote repositories via Git only when you explicitly enable it
  • Is sent to AI or search services only when you actively invoke them

Data Sharing

MeshWork does not proactively share your data with third parties. Data is sent to services you choose only when you explicitly perform one of the following actions:

  • You choose to push data to a remote repository via Git
  • You actively use AI features, sending content to your configured AI service provider
  • You actively use web search features, sending queries to your search provider

In these cases, data transfer occurs directly between your device and the services you choose. MeshWork does not access or store the transmitted content.

Analytics, Advertising, and Tracking

MeshWork does not include advertising SDKs, does not use third-party analytics SDKs, and does not track you across apps or websites. We do not sell your data.

Data Security

We take the following measures to protect your data:

  • API keys, Git tokens/passphrases/private key content, and proxy passwords saved by the app are stored in the macOS keychain
  • Non-sensitive settings such as service names, models, URLs, ports, usernames, and interface preferences are stored locally
  • The app runs within the macOS application sandbox, limiting file system access
  • Network communication is only used for services you explicitly configure

Children's Privacy

MeshWork is not directed at children under 13 years of age. MeshWork does not knowingly collect personal information from children.

Changes to This Privacy Policy

We may update this privacy policy from time to time. Updated policies will be posted on this page with the revision date noted at the top.

Contact Us

If you have any questions or suggestions about this privacy policy, please contact us:

  • Submit an Issue or Pull Request on GitHub
  • Contact the developer through the App Store product page
