MeshWork Manual

MeshWork is a creative workspace built around notes, discussions, AI, and Git sharing. This manual will help you systematically understand how to use MeshWork.

Core Concepts

∞ Workspace
The main usage environment users see when entering MeshWork. It's a product and interaction layer concept, not a fixed disk directory or Git repository. The workspace is responsible for combining one private library and multiple shared spaces.
📁 Private Library
The data boundary belonging to the current user. Suitable for storing personal long-term notes, research materials, writing drafts, inspiration fragments, and content that shouldn't automatically enter team sharing.
👥 Shared Space
A team or project-level data boundary. Each shared space contains a public library, discussion area, shared attachments, and can optionally have its own Git history.
💬 Discussion Area
An asynchronous discussion space within a shared space, supporting topic management, @ mentions, # document references, / commands, and AI agent collaboration.
🤖 AI Agent
In MeshWork, AI is a participant rather than an automatic replacement. It can suggest, but will not modify your documents without explicit confirmation.

Product Architecture and Knowledge Flow

MeshWork adopts a "private-first, progressive sharing" design philosophy, where knowledge flows from personal creation to team collaboration, ultimately settling into reusable long-term knowledge.

Knowledge Flow Path: Create and accumulate content in your private library, with AI providing intelligent assistance. When team collaboration is needed, you can directly share content to shared spaces: discuss in the discussion area, then valuable content further settles into the public library as long-term knowledge. AI provides assistance at every step, but won't automatically share your private content.
